Problem¶

With the yelp academic dataset, we wanted to provide restaurant recommendations to users in the dataset. We used three collaborative filtering methods to provide recommendations:

  • manhattan distance: \[d(x,y) = \sum^n_{k=1}|x_k-y_k|\]

  • euclidean distance: \[d(x,y) = \sqrt{\sum^n_{k=1}{|x_k-y_k|}^2}\]

  • cosine similarity: \[cos(x,y)=\frac{x\cdot y}{\|x\|\times\|y\|} \]

With the help of this website, we were able to implement these methods for our dataset.

To test the relative strength of each method, we calculate the root mean squared error for each user and method.

Our data was downloaded from here. It was in a .json format, so it was straightforward to do additional data gathering, cleaning and analysis in python. Afterwards, we filtered the dataset to only food-related reviews and businesses. Then we used the Yelp API to get missing state and city values for certain businesses. Challenges¶

Pearson Correlation Coefficient Method¶

Originally, we intended to implement one additional method, the Pearson Correlation Method on top of the existing methods we propose. However, the method proved to be ineffective at showing relationships amongst users from our dataset. This is primarily because of the limited data that we have such that upon evaluations using the Pearson Method mostly return 0. Therefore, we decided to scrap the method altogether and instead focus on using Cosine Similarity.

Parallelizing the Testing Framework¶

Since the dataset had more than 10,000 users, we wanted to take advantage of these numbers test our recommendation engine against the data that we have for these users. However, testing for each user sequentially would have taken more than 30 hours. Thus, we decided to parallelize our testing framework and acheived an 8x speedup on an standard multicore machine.

Results¶

Best Filtering Method¶

In order to check the quality of our predictions, we calculated root mean square errors for each user. To calculate the errors, we found the average of all the absolute differences between a user's predicted ratings for a restaurant and their actual rating. In order to compare the two, we act as if the user didn't go to the business in the first place, and then compare the predicted rating from our collaborative filtering and the actual value. Here's the formula for each user:

\[{rms\:error}=\sqrt{\frac{\sum{(actual\:rating-predicted\:rating)}^2}{n}} \] where \(n\) is the number of ratings a user has.
